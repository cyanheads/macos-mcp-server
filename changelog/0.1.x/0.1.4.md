# 0.1.4 — 2026-06-12

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.9` → `^0.10.6`
- **`createApp()`** now sets `name` and `title` explicitly to `macos-mcp-server` — display identity no longer derives from the npm-scoped package name
- **`format`** script split — `format` runs safe Biome fixes only; new `format:unsafe` keeps the `--unsafe` pass

## Added

- **`release:github`** script + `scripts/release-github.ts` — creates/repairs the GitHub Release on the version tag with the `v<version>: <subject>` title and `.mcpb` attach
- **`scripts/clean-mcpb.ts`** — wired into `bundle`; prunes dev dependencies and strips agent-doc trees (`skills/`, `.claude/`, `.agents/`, `SKILL.md`) nested under `node_modules/` that root-anchored `.mcpbignore` patterns cannot reach
- **`scripts/check-skill-versions.ts`** — devcheck step that fails a `skills/<name>/SKILL.md` body change unless `metadata.version` is bumped in the same edit
- **`.claude-plugin/`** and **`.codex-plugin/`** plugin manifests for marketplace install
- **`manifest.json`** — `repository`, `homepage`, and `license` fields

## Fixed

- **Error codes** corrected to semantic JSON-RPC codes: `not_running` (`macos_manage_apps`) and `finder_not_open` (`macos_manage_finder`) now return `NotFound`; `path_not_writable` (`macos_take_screenshot`) and the screencapture region/window/path guards now return `ValidationError` — previously all `InvalidParams`

## Removed

- **`tsx`** and **`esbuild`** dropped from the lockfile — no longer pulled in transitively
